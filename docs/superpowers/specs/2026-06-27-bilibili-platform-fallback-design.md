# B 站及平台视频解析后备设计

## 目标

让现有桌面下载器能够下载公开可访问的 B 站视频，包括音视频分离的 DASH 流，并保持原有下载按钮、直链、HLS 和央视页面下载能力不变。

验收链接：

`https://www.bilibili.com/video/BV1jL5F6PEog/`

## 方案

将 `yt-dlp` 作为平台解析后备集成到程序中。现有解析器仍是第一优先级；只有当页面中没有可用的下载按钮、直链、HLS 或已支持的播放器参数时，才调用 `yt-dlp`。

不手写 B 站接口签名，不自动读取 Chrome、Edge 等浏览器中的 Cookie，也不绕过登录、会员、付费、地区或 DRM 限制。

## 架构与数据流

1. `resolve_url()` 继续处理 MP4、M3U8 和普通网页。
2. 原有网页解析无法找到候选源时，返回一个 `platform-video` 类型的 `ResolvedVideo`，由 `yt-dlp` 负责最终解析。
3. `download_resolved_video()` 收到 `platform-video` 后调用独立的下载函数。
4. 平台下载函数使用 `yt_dlp.YoutubeDL`：
   - 选择最高可用质量的画面与音频；
   - 优先合并为 MP4；
   - 使用项目现有的 `imageio-ffmpeg` 可执行文件完成合流；
   - 输出文件仍遵守保存目录、自定义文件名和重名避让规则。
5. 下载结束后返回实际生成的文件路径，现有 GUI 继续显示完成信息并允许打开保存目录。

## 错误处理

- 未安装或未能打包 `yt-dlp`：提示程序组件缺失。
- 登录、会员、付费或地区受限：展示平台返回的可理解错误，不尝试读取私人 Cookie。
- 网络、提取或合流失败：转换为现有 `DownloadError`，避免 GUI 线程崩溃。
- 平台输出文件名发生变化：根据 `yt-dlp` 的下载结果定位实际文件，不凭模板猜测路径。

## 测试

- 单元测试先覆盖平台 URL 的后备分流。
- 单元测试覆盖 `yt-dlp` 参数、FFmpeg 路径、输出命名和错误转换。
- 运行现有完整测试，确保央视、直链和 HLS 行为没有回归。
- 使用指定 B 站链接实际解析并下载一个可播放的 MP4。
- 重新运行 PyInstaller 打包，验证桌面 EXE 可启动并完成平台下载流程。

## 非目标

- 不新增登录界面或 Cookie 管理。
- 不支持受 DRM 保护的视频。
- 不重做当前 UI，也不改变普通网页的下载优先级。
